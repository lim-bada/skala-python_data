# 창의적 개인 실습: 서비스 장애 조기탐지 및 원인 분석

## 1. 문제 설정

운영자는 수십만 건의 웹 로그를 직접 확인할 수 없습니다. 고정된 `5xx 오류율 10%` 같은 단일 규칙은 평소 트래픽 변화에 민감하고, 수많은 연속 알림 때문에 실제 장애 대응이 늦어질 수도 있습니다.

이 실습에서는 다음 문제를 해결합니다.

> 평소와 다른 트래픽·오류 패턴을 자동으로 찾아내고, 어떤 경로와 상태 코드가 장애에 가장 크게 기여했는지 설명할 수 있는가?

## 2. 이전 실습과 다른 점

이전 실습 1은 경로·상태·시간대별 개수를 집계했습니다. 이번에는 집계값을 머신러닝 특성으로 사용해 **정상 패턴을 스스로 학습하고 알려지지 않은 장애를 탐지**합니다.

새롭게 수행하는 내용은 다음과 같습니다.

- 정답 레이블 없이 학습하는 `IsolationForest` 비지도 이상 탐지
- 장애 전 구간만 정상 기준으로 사용하는 시계열 학습 분리
- 재현 가능한 503 오류·트래픽 급증 장애 시뮬레이션
- 시간 구간 단위 정밀도·재현율·F1 평가
- 연속된 이상 경보를 하나의 장애로 통합하는 알림 중복 제거
- 경로·상태 코드·IP별 장애 원인 후보와 초과 오류 수 계산
- 탐지 모델 저장·재로딩과 자동 HTML 장애 리포트

## 4. 장애 시나리오

기본 실행에서는 원본을 수정하지 않고 다음 장애를 메모리에서 재현합니다.

- 시간: `2025-03-05 14:00~14:30`
- 경로: `/api/v1/search`
- 현상: 해당 요청을 `503`으로 변경하고 트래픽을 5배로 증폭
- 정답: 5분 단위 장애 구간 6개

또한 Pydantic 검증 확인을 위해 잘못된 IP, `999` 상태 코드, 음수 전송량 로그를 각각 1건 추가합니다.

## 5. 단계별 실습

1. **STEP 0**: 데이터·설정·템플릿 확인
2. **STEP 1**: 청크 단위 로그 수집과 Pydantic 검증
3. **STEP 2**: 재현 가능한 503 장애 시나리오 생성
4. **STEP 3**: Polars 5분 단위 장애 특성 집계
5. **STEP 4**: 장애 이전 데이터로 IsolationForest 학습 및 탐지
6. **STEP 5**: 정밀도·재현율·F1 평가
7. **STEP 6**: 연속 경보 통합 및 원인·영향 분석
8. **STEP 7**: Parquet·JSON·Joblib 저장과 라운드트립
9. **STEP 8**: Plotly·Jinja2 HTML 리포트 생성

## 6. 실행 방법

프로젝트 최상위 폴더에서 실행합니다.

```bash
source .venv/bin/activate
python Advanced/advanced.py
```

장애 또는 오염 예시를 주입하지 않는 옵션도 제공합니다.

```bash

```

## 8. 산출물

`Advanced/output/`에 다음 파일이 생성됩니다.

- `anomaly_windows.parquet`: 5분 단위 특성·이상 점수·탐지 결과
- `incidents.json`: 중복 제거된 장애와 원인·영향 분석
- `invalid_logs.json`: 오염 로그 원본과 필드별 오류
- `isolation_forest.joblib`: 탐지 모델·임계값·특성 목록
- `incident_report_YYYYMMDD_HHMMSS.html`: 장애 시계열과 원인 분석 리포트

## 9. 성공 판정 기준

- 입력 건수 = 유효 건수 + 오염 건수이며 오염 예시 3건이 모두 탐지됨
- 원본 로그가 장애 시나리오 생성 과정에서 변경되지 않음
- IsolationForest가 장애 이전 정상 구간만 학습함
- 주입한 장애 6개 구간 중 90% 이상 탐지함
- 시간 구간 탐지 정밀도가 60% 이상임
- 주입 장애의 상위 오류 경로가 `/api/v1/search`, 상태가 `503`으로 분석됨
- 연속된 6개 장애 구간이 하나의 통합 경보로 중복 제거됨
- Parquet과 IsolationForest 저장 전후 결과가 일치함
- HTML에 Plotly 차트와 IP 마스킹 결과가 포함됨
- Pytest와 Ruff 검사를 모두 통과함

## 10. 해석 시 주의사항

- 주입 장애의 정답 구간은 평가에만 사용하며 모델 특성에는 포함하지 않습니다.
- 비지도 이상 탐지는 장애가 아닌 이벤트도 탐지할 수 있으므로 경보를 원인 확정으로 해석하면 안 됩니다.
- 현재 임계값은 정상 기준 이상 점수의 상위 0.1%입니다. 운영 환경에서는 경보 대응 비용에 맞춰 재조정해야 합니다.
- 상위 IP는 원인 후보가 아니라 추가 조사가 필요한 관찰 대상입니다.
